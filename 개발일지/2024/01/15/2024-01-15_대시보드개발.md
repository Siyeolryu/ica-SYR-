# 개발일지: "당신이 잠든 사이" 대시보드 개발

## 작성시각
2024년 1월 15일 (만든 날)

## 해결하고자 한 문제
사람들이 미국 주식 중에서 "지금 가장 핫한 주식"을 쉽게 볼 수 있는 웹사이트를 만들고 싶었어요. 마치 유튜브에서 인기 동영상을 보는 것처럼, 주식도 "오늘 가장 많이 거래된 주식"이나 "오늘 가장 많이 오른 주식"을 한눈에 볼 수 있게 만들고 싶었습니다.

## 해결된 것

### 1. 웹사이트 기본 틀 만들기
- 웹사이트를 만드는 도구(Next.js)를 설치하고 설정했어요
- 글자와 색깔을 예쁘게 꾸미는 도구(TailwindCSS)를 설정했어요
- 어두운 배경(다크 모드)으로 만들었어요
- 주식이 오르면 초록색, 내리면 빨간색으로 보이게 했어요

### 2. 화면에 보이는 것들 만들기
- **상단 메뉴**: 웹사이트 맨 위에 "당신이 잠든 사이"라는 제목과 메뉴를 만들었어요
- **주식 카드**: 주식 정보를 보여주는 카드 모양의 박스를 만들었어요
  - 큰 카드: 1등 주식을 크게 보여줘요
  - 작은 카드: 다른 주식들을 작게 보여줘요
- **브리핑 카드**: 만든 리포트들을 보여주는 카드를 만들었어요
- **버튼**: 클릭할 수 있는 버튼들을 만들었어요 (만들기 버튼, 보내기 버튼 등)

### 3. 페이지 2개 만들기
- **첫 번째 페이지 (메인 화면)**: 
  - 오늘 가장 핫한 주식 1등을 크게 보여줘요
  - 왜 이 주식이 1등인지 이유를 보여줘요 (예: "거래량 1위", "상승률 1위")
  - 예전에 만든 리포트들을 작은 카드로 보여줘요
  - "새 리포트 만들기" 버튼이 있어요
- **두 번째 페이지 (리포트 자세히 보기)**: 
  - 리포트에 들어있는 그림을 보여줘요
  - 리포트에 써있는 글을 보여줘요
  - 이메일이나 슬랙으로 보낼 수 있는 버튼이 있어요
  - 리포트에 포함된 주식들을 보여줘요

### 4. 테스트용 데이터 만들기
- 실제 주식 데이터 대신, 테스트용 가짜 데이터를 만들었어요
  - 예: 애플(AAPL), 테슬라(TSLA) 같은 주식 정보
  - 예: 예전에 만든 리포트 3개
- 숫자를 예쁘게 보여주는 기능을 만들었어요
  - 예: 1000000 → "1M" (백만)
  - 예: 185.5 → "$185.50" (달러 표시)

### 5. 모바일에서도 잘 보이게 만들기
- 핸드폰으로 봐도, 태블릿으로 봐도, 컴퓨터로 봐도 잘 보이게 만들었어요
- 작은 화면에서는 카드가 1줄로, 큰 화면에서는 3줄로 보여요

### 6. 사용자 경험 개선
- 버튼을 누르면 "처리중..."이라고 표시해요
- 마우스를 올리면 색깔이 바뀌어요
- 완료/처리중/실패 상태를 색깔로 구분해요
- 이메일/슬랙 아이콘을 보여줘요

## 해결되지 않은 것

### 1. 실제 주식 데이터 가져오기
- 지금은 가짜 데이터만 보여줘요
- 나중에 실제 주식 데이터를 가져오는 서버와 연결해야 해요
- 로그인 기능도 아직 없어요

### 2. 리포트 그림 만들기
- 리포트 그림은 지금은 빈 박스만 보여줘요
- 나중에 AI로 실제 그림을 만들어야 해요

### 3. 실시간으로 업데이트하기
- 지금은 한 번 보면 그대로예요
- 나중에 주식 가격이 바뀌면 자동으로 화면도 바뀌게 해야 해요

### 4. 사용자 설정
- 이메일 주소나 슬랙 주소를 저장하는 기능이 없어요
- "매일 아침 6시에 자동으로 보내기" 같은 기능도 없어요

### 5. 에러 처리
- 인터넷이 끊기거나 서버가 고장나면 어떻게 할지 아직 안 만들었어요
- 에러 메시지를 예쁘게 보여주는 기능이 없어요

### 6. 테스트
- 제대로 작동하는지 확인하는 테스트를 아직 안 만들었어요

## 향후 개발을 위한 컨텍스트

### 왜 이렇게 만들었는지
1. **Next.js Pages Router 사용**: 다른 사람들이 많이 쓰는 방식이라서 선택했어요
2. **TypeScript 사용**: 실수를 줄이기 위해 타입을 체크하는 도구를 썼어요
3. **색깔 설정**: 주식이 오르면 초록색, 내리면 빨간색으로 보이게 색깔을 정했어요
4. **파일 정리**: 비슷한 기능끼리 폴더에 모아놨어요

### 다음에 할 일 (우선순위 순서)

**1순위: 실제 데이터 가져오기**
- 실제 주식 데이터를 가져오는 서버와 연결하기
- 리포트를 만드는 서버와 연결하기
- 예전 리포트를 가져오는 서버와 연결하기

**2순위: 로그인 기능**
- 사용자가 로그인할 수 있게 만들기
- 로그아웃 기능 만들기

**3순위: 리포트 그림 만들기**
- AI로 리포트 그림을 만드는 서버와 연결하기
- 만든 그림을 저장하는 기능

**4순위: 사용자 설정**
- 이메일 주소 저장하기
- 슬랙 주소 저장하기
- 자동으로 보내는 시간 정하기

**5순위: 에러 처리와 테스트**
- 에러가 나면 예쁘게 보여주기
- 제대로 작동하는지 테스트하기

### 참고사항
- 지금은 모든 기능이 가짜 데이터로 작동해요
- 나중에 실제 서버와 연결할 때 `lib/mockData.ts` 파일의 함수들을 바꿔야 해요
- "브리핑 생성" 버튼과 "보내기" 버튼은 지금은 알림창만 띄워요
- 리포트 그림은 지금은 빈 박스만 보여줘요

### 파일 구조 (어디에 뭐가 있는지)
```
components/     - 화면에 보이는 것들 (카드, 버튼 등)
lib/           - 데이터와 계산하는 함수들
pages/         - 실제 화면들 (첫 페이지, 리포트 페이지)
styles/        - 색깔과 디자인 설정
```

### 쉽게 설명하면
이 프로젝트는 마치 "레고 블록"을 쌓아서 집을 만드는 것과 같아요. 
- 각 블록(컴포넌트)을 하나씩 만들고
- 블록들을 조립해서 방(페이지)을 만들고
- 방들을 연결해서 집(웹사이트)을 만든 거예요!

지금은 집의 뼈대는 다 만들었지만, 실제로 살 수 있게 하려면(실제 데이터를 가져오려면) 전기와 수도(서버 연결)를 설치해야 해요.
